MSF渗透Mysql

常用攻击模块

1、auxiliary/scanner/mysql/mysql_login

  • 常用于内网中的批量以及单主机的登录测试

  • mysql 的mysql.user 表的hash
  • 对应的sql语句：SELECT HOST,USER,PASSWORD FROM mysql.user

  • 常用于内网中的批量mysql主机发现

  • 常用于root启动的MySQL 并 root 的udf提权

6、exploit/windows/mysql/mysql_mof

  • mysql mof提权

7、exploit/windows/mysql/scrutinizer_upload_exec

  • 上传文件执行

MSF渗透SQL SERVER

MSF渗透SQL SERVER

msf常用模块

1、auxiliary/admin/mssql/mssql_enum

  • 获取sql server的信息

2、auxiliary/admin/mssql/mssql_enum_sql_logins

  • 枚举sql logins，速度较慢

3、auxiliary/admin/mssql/mssql_escalate_dbowner

  • 发现dbowner，当sa无法得知密码的时候，或者需要其他账号提供下一步的内网渗透

4、auxiliary/admin/mssql/mssql_exec

  • 当没有激活xp_cmdshell，自动激活。并且调用执行cmd命令。权限继承sql server

5、auxiliary/admin/mssql/mssql_sql

  • 指定sql server语句

6、auxiliary/admin/mssql/mssql_sql_file

  • 当需要执行多条sql语句的时候，或者非常复杂。msf本身支持sql文件

7、auxiliary/admin/mssql/mssql_hashdump

  • mssql的hash导出
  • 对应Sql语句：select password_hash from sys.sql_logins where name=’sa’

8、auxiliary/scanner/mssql/mssql_login

  • 可用来批量发现内网mssql主机

9、auxiliary/scanner/mssql/mssql_ping

  • 查询mssql实例

10、exploit/windows/mssql/mssql_payload

  • 针对不同版本的系统有独特的方式上传payload

11、post/windows/manage/mssql_local_auth_bypass

hacker101 CTF – Micro-CMS V2

Hints for “Micro-CMS v2”
Flag0 – Found

Regular users can only see public pages
Getting admin access might require a more perfect union
Knowing the password is cool, but there are other approaches that might be easier

Flag1 – Found

What actions could you perform as a regular user on the last level, which you can't now?
Just because request fails with one method doesn't mean it will fail with a different method
Different requests often have different required authorization

Flag2 – Found

Credentials are secret, flags are secret. Coincidence?

Payload分离免杀思路

Payload分离免杀思路

​ 目前的反病毒安全软件，常见有三种，一种基于特征，一种基于行为，一种基于云查杀。云查杀的特点基本也可以概括为特征查杀。无论是哪种，都是针对PE头文件的查杀。尤其是当payload文件越大的时候，特征越容易查杀。

​ 采取特征域行为分离面纱。避免PE头文件，并且分离行为，与特征的综合免杀。
